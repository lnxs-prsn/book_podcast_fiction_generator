> Generated by comparing `build_specsv10.md` against the current `src/` tree.

## Executive Summary

The v10 spec was implemented to a **high degree of completeness**. All major architectural pieces are present: the `PYTHONPATH=src` invocation model, package `__init__.py` files, shared config/exception/types/settings layers, engine adapters, podcast endpoints, the new `src/cli/podcast.py` and `src/cli/fiction.py` CLIs, the novel-pipeline callback refactor, and the shimmed legacy entry points.

The remaining gaps are **minor** and fall into three buckets:

1. A documentation file-location mismatch (`src/initial_readme.md` was specified but does not exist; the update landed in the project-root `initial_readme.md`).
2. Two legacy files inside `podcast_script_generator/llm/` (`parse_args.py` and `test_all.py`) still contain `print`/`sys.exit`, causing the spec's own "no prints / no sys.exit outside `__main__`" done-conditions to fail even though they were not explicitly listed for modification.
3. A few small behavioral discrepancies in `generate_book_podcast` / `generate_chapter_podcast` (sort order, validation ordering, parameter typing).

All Phase 8 smoke tests pass **except** three that depend on the items above.

### Critical gaps to fix

1. **`src/initial_readme.md` does not exist.** Either create it as a copy of the relevant section from the root `initial_readme.md`, or update the spec to point at the root file.
2. **`parse_args.py` still contains `sys.exit(1)` and `print(...)`.** Convert the realworld-mode error path to raise a typed exception (`ScriptGenerationError` or `ValueError`) so the spec's "no sys.exit outside `__main__`" invariant holds.
3. **`test_all.py` still contains prints/sys.exit.** These are test artifacts, but they cause the spec's grep-based done-conditions to fail. Either exempt test files from the condition or migrate them.
4. **`generate_book_podcast` should sort chapters numerically** to match the spec literally, e.g., by extracting leading digits from stems.
