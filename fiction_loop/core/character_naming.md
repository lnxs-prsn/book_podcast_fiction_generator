# CHARACTER NAMING — RULES + USED-NAME LEDGER

Canonical naming source for the Assembler (owner decision D7/F17). This file replaces
the naming note that previously lived in living_document.md's "NOTES FOR AI" section
(fragile there — that document is rewritten wholesale by an LLM every chapter).

The Assembler reads this file before generating any new character. The Updater appends
to the LEDGER section (STEP 8B) — the ledger is the ONLY part of this file an agent
may modify; the RULES section is owner-maintained.

---

## RULES

1. African names only — never Arabic names.
2. Names should be plausibly matched to the character's city and region (a Yoruba name
   in Lagos, an Akan name in Accra, a Kikuyu name in Nairobi, a Wolof name in Dakar,
   an Amhara name in Addis Ababa, a Shona name in Harare, a Zulu or Xhosa name in
   Johannesburg — mixing is realistic in big cities, but the default is local).
3. Never reuse a name from the LEDGER below, and avoid near-collisions (Amara/Amaka
   in adjacent chapters reads as a mistake).
4. Occupations are ordinary and unrelated to gate clearing (world_rules §1).
5. Cities are contemporary African cities. Vary them — the story is continental, not
   single-city (the anchor's omnipresence across cities is a planted clue, D3).
6. One fully improvised newcomer minimum per gate chapter (owner rule F15).
