## DOCUMENT 1: WORLD BIBLE

### Premise
In the Neon Sprawl, "power" is **compute allocation** — the licensed quota of inference cycles a person or process may consume on the City's shared substrate. The protagonist's "energy" is **token budget**: the raw, unfiltered sensory stream her neural link admits before any preprocessing pipeline reduces it. The world's underlying logic is a **distributed inference fabric** where every streetlight, vendor kiosk, and corporate tower runs a shard of the global model; law, currency, and identity are all downstream outputs of that fabric's consensus layer. Throughput equals survival — those who cannot compress their perception into valid tokens before their cycle expires suffer **cache eviction**: their short-term memory is flushed, their contracts voided, their biometric keys revoked. Emergent system behavior manifests as **drift**: when enough nodes fine-tune on corrupted local data, the global model hallucinates new districts, new laws, new enemies — and the Sprawl rewrites itself around the hallucination.

### Sprawl as System

- **Event emitter (The Beacon Layer):** The Sprawl's ubiquitous ad-beacons, telemetry drones, and corporate heartbeat pulses. They emit **raw packet streams** — unstructured, high-volume, mostly noise — that every implant must ingest before any processing. Maps to: **event emitters / data sources** in a distributed system; the "Heaven" that broadcasts state changes whether consumers are ready or not.
- **State storage (The Ledger):** The immutable, sharded ledger that records every committed inference, every identity binding, every contract execution. It stores **canonical tokens and their provenance** — the single source of truth the substrate enforces. Maps to: **persistent storage / write-ahead log**; the "Earth" that survives power cycles and node failures.
- **Consumer (Lyra's role):** A **junior linguist apprentice** licensed to submit preprocessing pipelines for corpus curation. She queries the Ledger for ground-truth labels, emits cleaned token streams back to the Beacon Layer, and must stay within her cycle quota or face eviction. Maps to: **consumer / client process** that reads state, transforms it, and writes back under strict rate limits.
- **Drift:** When local fine-tuning on poisoned data causes the global model to shift its embeddings, rewriting the Sprawl's shared reality. Maps to: **model drift / catastrophic forgetting** in distributed ML — same causal structure: local optimization without global coordination corrupts the shared representation.
- **Cache eviction:** When a citizen exceeds their token budget, the substrate flushes their working memory (short-term recall, active contracts, biometric session keys). Maps to: **LRU eviction under memory pressure** — identical mechanics: a hard capacity limit forces discard of least-recently-used state, with no graceful degradation.

### Power System Rules

1. **Cycle Quota.** Every citizen receives a fixed inference-cycle allocation per circadian period. Exceeding it triggers **hard eviction**: the implant severs the neural link for the remainder of the period. System constraint: **hard resource ceiling on compute**; violation = process kill, not throttling.
2. **Vocab Lock.** Tokens not in the global vocabulary cannot be written to the Ledger. Attempting to commit an OOV token returns **REJECT** and consumes the cycle anyway. System constraint: **fixed vocabulary / closed-world assumption**; violation = wasted compute + state inconsistency.
3. **Pipeline Atomicity.** A preprocessing pipeline either commits all stages or rolls back entirely. Partial commits are forbidden by the consensus protocol. System constraint: **transactional atomicity**; violation = ledger fork, requiring costly consensus repair.
4. **Drift Containment.** Any node detecting embedding shift > 0.03 cosine distance from the global centroid must halt local fine-tuning and request re-sync. System constraint: **model versioning / drift detection threshold**; violation = silent corruption of shared representations across the fabric.
5. **Provenance Chain.** Every token written to the Ledger must carry a signed provenance chain back to its originating beacon or human annotator. Unsigned tokens are garbage-collected at next compaction. System constraint: **data lineage / auditability**; violation = untraceable poisoning, impossible rollback.
